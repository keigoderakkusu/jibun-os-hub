## 🔄 データフロー図

### パターン A: 物理書類スキャン（単一ページ）

```
書類
 │
 ├─ スマートフォンカメラで撮影
 │   👁️ (実物の書類 → デジタル画像)
 │
 ▼
PNG 画像（2-5 MB）
 │
 └─ Base64 エンコード
     (テキスト形式に変換)
 │
 ▼
HTTP POST → GAS API
 │
 ▼
Base64 デコード → PNG Blob
 │
 ▼
Google Document 生成
 │
 │ (Google Docs が自動 OCR)
 │
 ▼
PDF エクスポート
 │
 ▼
Google Drive に保存
 │
 ▼
✅ アクセス可能な PDF
   (Google Drive のリンク)
```

**処理時間**: 30-60 秒（ファイルサイズ依存）

---

### パターン B: 物理書類スキャン（複数ページ）

```
書類ページ 1  書類ページ 2  書類ページ 3
     │             │             │
     └─────┬───────┴─────┬───────┘
           │             │
           ▼ スマートフォンで撮影
         PNG 画像配列
         [img1, img2, img3]
           │
           └─ 各画像 Base64 エンコード
             │
             ▼
HTTP POST → GAS API (images: [...])
             │
             ▼
For Each Image → Google Document に挿入
             │
             ├─ イメージ 1
             ├─ ページブレーク
             ├─ イメージ 2
             ├─ ページブレーク
             ├─ イメージ 3
             │
             ▼
PDF エクスポート
(3 ページの連続 PDF)
             │
             ▼
Google Drive に保存
             │
             ▼
✅ 3 ページの連続 PDF
   (1 ファイル内)
```

**処理時間**: 60-120 秒（ページ数・ファイルサイズ依存）

### パターン C: 自動スクロール連続スクショ

```
URL: https://example.com/article
           │
           ▼
Puppeteer ブラウザ起動
           │
           ▼
ページ読み込み (networkidle2)
           │
           ▼
スクロール戦略を選択
           │
     ┌─────┼─────┐
     │     │     │
     ▼     ▼     ▼
 Full  Viewport  Paginated  無限スクロール
     │     │     │         │
     │     ▼     │         │
     │   スクロール ▼         │
     │   500ms待機   次ボタン →  スクロール
     │     │       クリック    終了検出
     │     │         │         │
     └─────┼─────────┼─────────┘
           │
           ▼ (複数回スクショ)
    PNG 画像配列
    [page1.png, page2.png, ...]
           │
           └─ Base64 エンコード（配列）
             │
             ▼
HTTP POST → GAS API (images: [...], pdfMode: "combined")
             │
             ▼
      PDF 生成エンジン
             │
        ┌────┴────┐
        │          │
        ▼          ▼
    Combined    Split
   (1 つの PDF)  (複数 PDF)
        │          │
        ▼          ▼
   PDF ファイル   複数 PDF ファイル
        │          │
        └────┬─────┘
             │
             ▼
Google Drive に保存
             │
             ▼
✅ PDF 作成完了
```

**処理時間**: 
- Full: 10-20 秒
- Viewport: 30-60 秒（ページ数に応じて）
- Paginated: 60-180 秒
- 無限スクロール: 30-120 秒

## 📈 スケーラビリティ

### 単一スクリーンショット処理

```
┌─────────────────────────────────────┐
│ GAS Web API（無制限サーバー）      │
│                                     │
│ 同時接続: 複数ユーザー対応         │
│ 処理能力: スケーラブル             │
│ ストレージ: Google Drive 無制限    │
└─────────────────────────────────────┘

スケーリング:
- 1 ユーザー: 60 秒で処理
- 10 ユーザー: 各々 60 秒（並列処理）
- 100 ユーザー: キューイング対応可能
```

### バッチ処理（複数 URL）

```bash
# 複数 URL を順序実行
for url in urls.txt; do
  node auto-scroll-screenshot.js --url "$url"
done

# 時間: URL 数 × 60 秒（目安）
```

### クラウド実行（Google Cloud Scheduler）

```
定時実行設定
    ↓
Cloud Functions / Cloud Scheduler
    ↓
複数 URL の自動処理
    ↓
PDF を自動生成 → Google Drive に保存
    ↓
メール / Slack 通知
```
